# ADR-\_: Реестр шагов SAGA

## Контекст

Основной сценарий платежа:

1.  Пользователь оформляет заказ и инициирует платёж.
2.  Система списывает деньги с его счёта.
3.  Выполняется проверка в антифрод-системе и внешних сервисах.
4.  Если проверка пройдена успешно, деньги переводятся контрагенту.
5.  Если операция признаётся подозрительной — она блокируется, клиенту мгновенно возвращаются деньги, а в систему безопасности уходит уведомление для анализа.

Проверка платежей Сервис антифрод-проверок может возвращать три типа ответа:

- Разрешение — транзакция может быть проведена.
- Запрет — транзакция отклоняется.
- Ручная проверка —
  - транзакция направляется на рассмотрение оператору, который должен принять решение в течение 20 минут. По результатам ручной проверки сервис возвращает одно из окончательных решений:
    - разрешение
    - или запрет.
  - На время ручной проверки транзакция должна находиться в состоянии ожидания.

## Проблема

Декомпозировать процесс на атомарные транзакции и классифицировать их по типам (компенсируемая / необратимая / повторяемая), обеспечив автоматическое восстановление или откат системы.

## Решение

|  |  |  |  |
|----|----|----|----|
| Транзакция | Описание | Тип | Компенсация |
| ———————– | ————————————————————- | ——————————————– | ————————————– |
| (в формате кода) | (Описание шага в свободной форме) | (компенсируемая / необратимая / повторяемая) | (наименование шага из первого столбца) |
| CREATE<sub>ORDER</sub> | Сохраняется объект платежа | компенсируемая | CANCEL<sub>ORDER</sub> |
| HOLD<sub>CLIENT</sub>\_FUNDS | Резеривирование средств на счету клиента | компенсируемая | RELEASE<sub>CLIENT</sub>\_FUNDS |
| VALIDATE<sub>PAYMENT</sub>\_AUTO | Автоматическая проверка на мошенничество | повторяемая |  |
| VALIDATE<sub>PAYMENT</sub>\_MANUAL | Ручная проверка на мошенничество | повторяемая |  |
| FINALIZE<sub>PAYMENT</sub> | Средства переводятся продавцу | необратимая |  |
| NOTIFY<sub>CLIENT</sub> | Оповещение клиента о статусе | повторяемая |  |
| NOTIFY<sub>SAFETY</sub>\_SYSTEM | Оповещение системы безопасности о подозрении на мошенничество | повторяемая |  |
| CANCEL<sub>ORDER</sub> | Компенсация. Смена статуса заказа на Отклонен | повторяемая |  |
| RELEASE<sub>CLIENT</sub>\_FUNDS | Компенсация. Снятие резервирования средств со счета клиента | повторяемая |  |

## Аргументация

- Pivot-точка:
  - Транзакция FINALIZE<sub>PAYMENT</sub> выбрана как точка невозврата. После её успешного выполнения автоматическая компенсация невозможна, процесс считается успешно завершенным с точки зрения бизнеса.
- Повторяемость:
  - Все шаги проверок и уведомлений сделаны повторяемыми (retryable) для обеспечения отказоустойчивости при временной недоступности внешних API.

## Ограничения и риски

- Идемпотентность:
  - Все повторяемые транзакции должны быть строго идемпотентны, чтобы избежать дублирования списаний или уведомлений при повторных попытках.
- Таймауты:
  - Ручная проверка ограничена временем ожидания. Превышение лимита требует настройки cut-off стратегии (автоматическое разрешение/отклонение).
